Worth to know:

Before using the plugin, you should know some of its behavior/bugs that may be the reason for not using this library at this time. Problems will be solved as a way is found and the author has free time. If there is a problem in this section, then it has not been solved yet :)

Disappearance:

The plugin uses the requestAnimationFrame function to run animation frames, it's a great feature, and it has a moment when the user moves from the tab on which the tape is to any other and stay on another tab for a few seconds, causes the function to stop to save resources. but it affects the tape in which more than one element, they just disappear, and this is a significant problem.

This problem will be solved in the near future.

To see the problem, switch to another tab for 10 seconds.

Please reload the page to read further.

Timing functions:

At this time, time functions do not work with multiple elements, which is a problem whose root is that when calculating motion and position, the elements know nothing about the fact that there is another element in the neighborhood.

Offset:

At set speeds of less than 5000, and even with some iterations of the tape, it can be shifted both to the right and to the left when calculating the path.

Also:

If you know how to solve the problems above, or you find another bug, you can:

  • Create a pull request with changes.
  • Write about a bug or suggestion in issues.
